Структура delivery bot.

Файли та папки
db -> config.php -> дані для підключення бази даних.
db -> db_functions.php -> функції для роботи з базою.
db -> db.php -> підключення до бази даних.

callback_worker.php -> обробник кнопок для робітника.
callback_buyer.php -> обробник кнопок для покупця.
deliveryhook.php -> основний обробник на який посилається telegram bot.
keyboard_worker.php -> inline кнопки для працювника 
keyboard_buyer.php -> inline кнопки для покупця.
telegram_api.php -> підключення до telegram bot.
token.php -> $TOKEN = 'botTOKEN'

Один бот - дві варіації. Працівник і покупець.
Чат в боті повинен бути як одне повідомлення яке оновлюється щоб чат не стрибав наверх.

База даних: tredmark_delivery_bot.
Таблиці:
users - id, telegram_id, username, role, balance, worker_status, created_at.
	role - worker, buyer
	worker_status - online, offline
regions - id, name
products - id, name, price
orders - id, buyer_id, product_id, region_id, assigned_worker_id, price, status, created_at
	assigned_worker_id - id працівника який взяв замовлення в роботу.
	status - searching_workers, assigned, in_progress, completed, canceled, timeout.

order_logs - id, order_id, worker_id, buyer_id, status, created_at.


Принцип роботи:
Користувач вперше запустив в бота і його потрібно записати в таблицю users з відповідними полями.
Статус при першому запуску завжди buyer. Адміністратор вручну назначає робітників по telegram_id в таблиці.

Текст - 
Привіт {username}, вітаєм в нашому магазині.

Кнопки які будуть показані -
Профіль
Поповнення
Міста 

При натисканні на Профіль буде показуватись базовий текст:
Name: {username}
Id: {telegram_id}

Кнопки:
Мої замовлення
Назад (повернення в головне меню)

Кнопка Мої замовлення:
Показує текстом 
Id замовлення: {id ордера з таблиці orders}
Регіон: {name щ таблиці regions по region_id з таблиці orders}
status: {status з таблиці orders}

Кнопка
Назад (повернення в головне меню)

Кнопка поповнення
Поки не активна і показати текст - В розробці
Кнопка Назад (повернення в головне меню)

Кнопка Міста - 

Текст - Виберіть потрібний регіон:
Кнопки з регіонами з таблиці regions.

Після кліку по потрібному регіону видавати:
Тексе - виберіть товар
Кнопки зі списком продуктів з таблиці products.
Вид кнопки - {name} {price}

Після натискання кнопку з продуктом
Текст - Замовити товар
Кнопка Замовити
Кнопка Назад (повернення в головне меню)

Після натискання кнопки замовити записати товар в таблиці orders з відповідними полями.
Статус searching_workers.

Для робітника.
Коли адмін поставить статус worker то для такого користувача буде інше меню
Головне меню:
Текст - На роботі / не на роботі.
Кнопки:
Почати роботу / закінчити роботу. online/offline в таблиці users для role - worker.
Вільні замовлення
Замовлення в роботі
Виконані замовлення

Вільні замовлення:
Показуються всі замовлення зі статусом searching_workers.
Кнопки позначені - id | region | price

При натисканні на таке замовлення буде текст - 
Замовлення id | region | price
Кнопка - Взяти замовлення.
Після натискання на кнопку це замовлення переться цим робітником і статус замовлення змінюється 
на assigned. В поле assigned_worker_id призначається саме цей робітник. Іншим це замовлення вже не доступне.
Також замовлення яке взялось в роботу попадає в таблиці orders_logs з відповідними полями, 
id, order_id, worker_id, buyer_id, status - assigned.

Кнопка замовлення в роботі:
показує які замовлення в даного робітника зараз в роботі з таблиці orders.

Кнопка виконані замовлення показує текст:
Виконані замовлення: {кількість виконаних замовлень з таблиці orders_logs}
